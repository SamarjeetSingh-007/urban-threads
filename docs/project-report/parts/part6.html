Testing

11.1 Testing Methodology

Testing is a critical phase of the Software Development Life Cycle (SDLC) that ensures the application meets its functional and non-functional requirements. For Urban Threads, both Black-Box Testing (testing functionality without knowledge of internal code) and White-Box Testing (testing with knowledge of internal code logic) approaches were used.

The following testing levels were applied:

  • Unit Testing: Testing individual functions and API endpoints in isolation to verify correct behavior.
  • Integration Testing: Testing interactions between frontend, backend, and database to ensure they work together correctly.
  • System Testing: End-to-end testing of complete user flows from registration to cart checkout.
  • User Acceptance Testing (UAT): Testing by classmates and friends to validate usability and gather user feedback.

11.2 Unit Testing

Authentication Tests:

  • Verified bcrypt password hashing and comparison functions work correctly with salt round 10
  • Verified JWT token generation produces valid tokens with correct payload (userId, email, role) and expiry (24 hours)
  • Verified expired and tampered tokens are correctly rejected by the authentication middleware
  • Verified registration endpoint rejects empty fields, invalid email formats, and passwords shorter than 6 characters
  • Verified duplicate email registration returns 409 Conflict status

Product API Tests:

  • GET /api/products returns complete product array with all fields populated
  • Category filter parameter returns only matching products (e.g., ?category=hoodies returns only hoodies)
  • Search parameter performs case-insensitive matching (e.g., ?search=urban matches "Urban Street Hoodie")
  • Product creation without admin token returns 403 Forbidden
  • Product creation with admin token and valid data returns 201 Created with the new product

Cart API Tests:

  • Add to cart creates new record when product+size combination doesn't exist
  • Add to cart increments quantity when product+size combination already exists in cart
  • Get cart returns only the authenticated user's items (not other users' carts)
  • Delete cart item removes the item and returns 200 OK
  • Operations without authentication token return 401 Unauthorized

11.3 Integration Testing

  • Login form correctly sends credentials to /api/auth/login and stores received JWT token in localStorage
  • Product catalog page fetches products from /api/products and renders them in the grid layout
  • Cart updates in real-time when items are added, quantity is changed, or items are removed
  • Category filter buttons send the correct query parameter and update the product grid dynamically
  • Foreign key constraints (ON DELETE CASCADE) prevent orphaned cart and wishlist items when users or products are deleted
  • Admin panel correctly communicates with admin-only endpoints using the admin JWT token
  • Image upload through admin panel correctly stores files in Supabase Storage and returns public URLs
  • Contact form submission stores messages in the database and displays success/error feedback to users

Chapter 12
Conclusion

The development of Urban Threads — a premium streetwear e-commerce platform — has been a comprehensive and rewarding project that demonstrates the practical application of full-stack web development skills acquired during the BCA program at Netaji Subhash University, Jamshedpur.

The project successfully achieved all its primary objectives. A fully functional e-commerce website was designed, developed, and deployed, providing users with a complete online shopping experience for streetwear fashion. The application features secure user authentication with bcrypt password hashing and JWT-based session management, a comprehensive product catalog with category filtering and search functionality, shopping cart and wishlist features with persistent storage across sessions, a contact form for user communication, and a built-in admin panel for complete content management.

The technical implementation demonstrates proficiency in multiple web technologies. The frontend was built using HTML5, CSS3, and vanilla JavaScript, resulting in a fast and responsive user interface with a premium dark-themed design. The backend was developed with Node.js and Express.js, following RESTful API design patterns with proper middleware architecture. PostgreSQL via Supabase provides reliable data storage with proper schema design, normalization to Third Normal Form (3NF), and referential integrity through foreign key constraints.

The application was successfully deployed on Vercel, demonstrating knowledge of modern cloud deployment practices including environment configuration, serverless architecture, continuous deployment from GitHub, and CDN distribution. The deployment ensures high availability, automatic HTTPS, and zero-downtime deployments.

Through the development process, several valuable lessons were learned about real-world software development challenges, including CORS configuration for cross-origin requests, environment variable management across development and production environments, secure authentication flows with proper password hashing, database query optimization using Supabase's JavaScript client, file uploads in serverless environments using Supabase Storage, and responsive design testing across diverse devices and screen sizes.

The project also demonstrated that a professional, feature-rich e-commerce platform can be built and deployed using entirely free and open-source tools. The total development cost was zero rupees, proving that aspiring entrepreneurs and small businesses can establish a strong online presence without significant financial investment.

While the current version has certain limitations — notably the absence of payment gateway integration, order tracking, social login, and a native mobile application — these are identified and documented as areas for future enhancement. The modular architecture and clean API design make it straightforward to add these features incrementally without requiring major refactoring.

In summary, Urban Threads serves as both a functional e-commerce platform and a comprehensive demonstration of full-stack web development capabilities. The successful completion of this project validates the approach of combining academic learning with practical application, and provides a strong portfolio piece for future career opportunities in software development.

Lessons Learned

  1. CORS Configuration: Understanding and properly configuring Cross-Origin Resource Sharing was one of the first real-world challenges encountered. The difference between development (permissive CORS) and production (restricted origins) was an important lesson in security.
  2. Environment Management: The importance of separating configuration from code using environment variables became clear when deploying to Vercel. Managing different configurations for development and production environments is a critical skill.
  3. Security First Approach: The critical importance of never storing plain-text passwords, always validating user input on the server side (never trusting client-side validation), and using short-lived JWT tokens for session management.
  4. Responsive Design: Using CSS Grid and Flexbox effectively for layouts that work across all devices. Testing on real devices revealed issues that desktop browser simulation missed.
  5. Cloud Deployment: Understanding serverless architecture and how Vercel converts Express.js servers into serverless functions. The concept of cold starts and their impact on API response times.
  6. Version Control: Using Git and GitHub for systematic code management. Writing meaningful commit messages and using branches for feature development proved invaluable for tracking changes.
  7. Database Design: The importance of proper schema design, normalization, and indexing for performance. Using UUIDs instead of auto-incrementing integers for primary keys improved security by making IDs unpredictable.

Chapter 13
Future Scope

Urban Threads has been designed with extensibility in mind. The modular architecture, clean API design, and well-structured database schema make it straightforward to add new features incrementally. The following enhancements are planned for future versions of the platform:

13.1 Payment Gateway Integration

Integrating Razorpay payment gateway for the Indian market, supporting UPI (Unified Payments Interface), credit and debit cards, net banking, and popular wallets like Paytm, PhonePe, and Google Pay. An alternative integration with Stripe can be added for international payments, enabling customers outside India to make purchases using international credit cards and Apple Pay.

13.2 Order Management System

A complete order lifecycle management system including order placement with payment confirmation, real-time status tracking through stages (placed, confirmed, packed, shipped, out for delivery, delivered), shipping integration with Indian logistics partners like Shiprocket and Delhivery for automated shipment booking, automated email and SMS notifications at each order status change, and order history with invoice generation and download capability.

13.3 User Reviews and Ratings

A star-based rating system (1-5 stars) with text reviews, image upload capability for user-submitted product photos, verified purchase badges to distinguish genuine reviews, review moderation by admin before publishing, aggregate ratings displayed on product cards in the catalog, and sorting products by average rating or number of reviews.

13.4 Social Authentication

OAuth 2.0 integration for convenient login through existing social accounts: Google Sign-In, Facebook Login, Apple Sign-In (for iOS users), and GitHub Login (for developers). This reduces registration friction and improves conversion rates by allowing users to sign up with a single click.

13.5 Mobile Application

Native mobile apps for Android and iOS using React Native or Flutter framework, sharing the same backend API. Mobile-specific features would include push notifications for order updates and promotions, biometric authentication (fingerprint, Face ID), camera-based product search using image recognition, offline browsing of previously viewed products, and app-exclusive deals and early access to new arrivals.

13.6 AI-Powered Recommendations

Machine learning algorithms for personalized product recommendations based on browsing history, purchase patterns, and collaborative filtering. The system would analyze user behavior to suggest products that match their style preferences, display "Customers who bought this also bought" sections, and create personalized homepages showing products tailored to each user's taste.

13.7 Multi-Language Support

Internationalization (i18n) for Hindi, Bengali, Tamil, Telugu, and other regional Indian languages. This includes translated UI elements, product descriptions, and checkout flow. The system would auto-detect the user's browser language and offer the appropriate translation, with a manual language selector in the navigation bar.

13.8 Progressive Web App (PWA)

Converting the web application into a Progressive Web App for offline access through service workers, push notifications for order updates and promotions, app-like installation from the browser without requiring app store downloads, and improved performance through asset caching. PWA support would significantly improve the experience for users on slow or unreliable internet connections.

13.9 Advanced Analytics Dashboard

An enhanced admin analytics dashboard with sales trend visualization (daily, weekly, monthly revenue charts), popular products ranking by views, cart additions, and purchases, user demographics analysis (location, device, browser), cart abandonment rate tracking with recovery email automation, revenue reports with export functionality (CSV, PDF), and integration with Google Analytics 4 for comprehensive web analytics.

13.10 Inventory Management

Automated stock tracking with low-stock alerts when product quantities fall below configurable thresholds, size-wise stock management, supplier integration for automated reordering, and stock history logging for audit purposes.
